# PhD-Research 使用指南

> 基于 **OpenCode** 的 AI 教育研究助手工作区。
> 多智能体文献检索、论文精读、研究创意生成 + 跨模型对抗审计 + 博士级方法论硬约束。

---

## 三件套

| 工具 | 角色 |
|------|------|
| **OpenCode** (本目录) | 命令与子代理引擎（执行模型 `claude-opus-4.7`，审计模型 `gpt-5`） |

## 命令速查

### 日常
| 命令 | 用途 |
|------|------|
| `/daily` | 每日例程：今日论文推荐 + 进度跟踪 |
| `/weekly-report` | 周报 |

### 文献流
| 命令 | 用途 |
|------|------|
| `/search-papers {主题}` | 多源检索（默认 5 年内）+ 自动 coverage 与 citation 审计 |
| `/summarize {DOI/标题}` | 深度笔记 + summary-auditor 审计 |
| `/lit-review {主题}` | 系统综述；加 `--mode=quick-survey` 限近 2 年综述 |
| `/concept {术语}` | 概念卡 + concept-auditor |
| `/fetch {ref}` | DOI/arXiv/URL → 自动入 Zotero |
| `/extract {paper}` | 论文表格/图数据结构化 |
| `/curate` | Zotero 库去重补全 |

### 研究核心
| 命令 | 用途 |
|------|------|
| `/brainstorm {主题}` | 小分支模式创意生成 + So-What Gate |
| `/theory-map {主题}` | 理论框架使用频次 + 关系图 |
| `/deep-dive {主题}` | 经典 6 阶段验证管道 |
| `/phd-route {主题}` | **博士级研究定位**（S1→S5 完整方法论） |
| `/draft {章节}` | 笔记 → 写作草稿 |

### 系统
| 命令 | 用途 |
|------|------|
| `/eval {agent}` | 基准回归测试 |
| `/meta-optimize` | 系统自演化建议（**需人工 approve**） |

所有命令支持 `--effort=lite|balanced|max|beast` 与 `--no-audit` 开关。

## 博士方法论硬约束（核心）

> **博士 = 在大家都研究的点上做一个小分支 + 产生理论贡献 + 回答 So What。**

### 5 步路径（S1–S5）

| 步骤 | 动作 | 目的 |
|------|------|------|
| **S1 深度检索** | AI 分析近 5 年文献 | 找热点 + gap |
| **S2 综述加速** | AI 找近 2 年综述 | 低成本看 SOTA |
| **S3 理论盘点** | 提取理论使用情况 | 找理论空白 |
| **S4 小分支定位** | 在主流热点下选小切口 | **大方向不创新，小分支创新** |
| **S5 So What 论证** | 必须有理论贡献，不仅是解决实际问题 | 过博士门槛 |

### 必填四字段（缺一即被拒）

每个研究 idea 必须给出：
- `mainstream_anchor` — 挂靠哪个主流方向
- `sub_branch` — 在该方向下的小切口
- `theoretical_contribution` — 对哪个理论做了扩展/挑战/整合
- `so_what` — 假如结论成立，谁会因此改变什么

### 反模式（系统会警告）
- ❌ "我要开创全新方向"
- ❌ "我解决了一个真实问题"（仅工程贡献）
- ❌ "这个 gap 没人做过"
- ❌ 不锚定任何理论框架

详见 `.opencode/memory/phd-doctrine.md`。

## 跨模型对抗审计

执行层用 `claude-opus-4.7`，审计层用 `gpt-5`，强制异构第二意见，避免同源偏见。

每次审计产生的 prompt + response 全保存到 `.opencode/traces/YYYY-MM-DD/`，可供 `/meta-optimize` 分析。

每条命令默认带审计后处理（`post_hooks`），可加 `--no-audit` 跳过。

| 理论谱系 | `Concept Cards/Theory Map - {topic}.md` |
| 写作草稿 | `Writing Drafts/{Title}.md` |
| 博士定位 | `Research Wiki/Routes/YYYY-MM-DD-{topic}.md` |

## 典型工作流

```
早上    /daily                           今日推荐
        /summarize {论文}                 深读 + 自动审计
新主题  /phd-route {主题}                 走完 S1-S5 找定位
深入    /lit-review {子分支}              系统综述
写作    /draft {章节}                    生成草稿
周五    /weekly-report                   回顾
不定期  /meta-optimize                   看系统改进建议
```
