这是当前项目的中文主说明。

## 现在这个项目是什么

这是一个已经去掉旧包袱后的 OpenCode 研究工作区。

保留的核心只有：

- `.opencode/` 运行层
- `domains/ai-in-education/` 领域知识层
- `docs/` 文档层
- `opencode.json` 配置层

旧的初始化器、模板系统、旧技能树、旧评测脚手架、旧缓存和旧历史文档已经从主架构中移除。

## 当前 7 个命令

| 命令 | 用途 |
|---|---|
| `/find` | 检索论文、解释概念、查已有笔记 |
| `/read` | 抓取并精读论文 |
| `/think` | 研究创意、概念卡、理论图谱 |
| `/write` | 综述、章节、草稿、回复信 |
| `/review` | 日/周/月研究回顾 |
| `/plan` | 博士级研究路线与 deep-dive 规划 |
| `/admin` | 系统维护 |

## `/admin` 是干嘛的

`/admin` 现在只保留两个实际子命令：

1. `meta-optimize`
2. `health`

### `/admin meta-optimize`

作用：

- 审查当前系统有没有漂移
- 看 traces、memory、verifier 结果
- 如果发现问题，往 `.opencode/proposals/` 里写 proposal

它不会直接改代码。

### `/admin health`

作用：

- 做系统健康检查
- 确认结构、自检脚本、关键目录是否正常

它是管理命令，不是研究命令。

## 模型配置

当前我已经把根模型改成更合理的默认值：

| 层 | 模型 |
|---|---|
| 根会话默认模型 | `github-copilot/claude-sonnet-4.6` |
| 重型代理 | `github-copilot/claude-opus-4.7` |
| 中型代理 | `github-copilot/claude-sonnet-4.6` |
| 轻型代理 | `github-copilot/claude-haiku-4.5` |
| 审计代理 | `github-copilot/gpt-5.4` |
| 审计回退 | `github-copilot/claude-opus-4.7` |

这样调整的原因：

1. 日常路由和调度不需要一直用最重的 Opus
2. Sonnet 更适合做默认主控，质量和速度更平衡
3. Opus 保留给长综述、深度综合、理论映射这类重活
4. GPT-5.4 继续保留做异构审计

## 现在保留的主要目录

### `.opencode/`

项目真正的核心运行层。

里面主要有：

- `command/`：7 个命令定义
- `agent/`：子代理定义
- `memory/`：长期记忆
- `plugins/`：插件
- `verifiers/`：自检规则
- `traces/`：运行轨迹
- `checkpoints/`：deep-dive 断点
- `proposals/`：系统改进提案

### `domains/`

当前保留的是：

- `domains/ai-in-education/`

它提供：

- 关键词映射
- 期刊层级
- 理论框架
- 方法清单
- 主题清单
- 社会议题

### `docs/`

精简后的项目说明文档。

### `scripts/`

维护脚本。
