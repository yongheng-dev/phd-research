# Scholar Flow 使用指南（中文版）

## 这是什么？

**Scholar Flow** 是一个基于 Claude Code 构建的 AI 驱动学术研究助手，专为你的教育技术方向 PhD 研究定制。它将 Claude Code 的 AI 能力与 Obsidian 知识库、Zotero 文献管理无缝整合。

---

## 工具生态三件套

| 工具 | 角色 |
|------|------|
| **Claude Code** (本目录) | 核心 AI 引擎，执行所有命令 |
| **Obsidian** (`/Users/xuyongheng/Obsidian-Vault`) | 知识库，自动保存所有笔记 |
| **Zotero** | PDF 存储 + 引用管理 |

---

## 核心命令速查

在 Claude Code 中直接输入斜杠命令：

### 日常使用
```
/daily
```
每日研究例程：推荐最新论文、跟踪研究进度。**每天开始工作时运行。**

---

### 文献搜索
```
/search-papers {主题}
```
多源学术搜索（Semantic Scholar + arXiv），自动生成搜索矩阵。

**示例：**
```
/search-papers AI literacy in higher education
/search-papers self-regulated learning with LLMs
```

---

### 论文精读
```
/summarize {DOI 或论文标题}
```
深度结构化阅读笔记，**自动保存到 Obsidian**。

**示例：**
```
/summarize 10.1016/j.compedu.2023.104xxx
/summarize "ChatGPT in higher education: a systematic review"
```

---

### 研究创意生成
```
/brainstorm {主题}
```
使用**碰撞矩阵法**系统性生成研究方向。可用于论文选题头脑风暴。

**示例：**
```
/brainstorm AI feedback in writing education
/brainstorm adaptive learning systems for K-12
```

---

### 系统文献综述
```
/lit-review {主题}
```
按 PRISMA 框架构建系统性文献综述。

**示例：**
```
/lit-review AI tutoring systems 2020-2025
```

---

### 概念卡片
```
/concept {术语}
```
生成陌生概念的解释卡片并保存到 Obsidian。

**示例：**
```
/concept cognitive load theory
/concept Zone of Proximal Development
```

---

### 周报
```
/weekly-report
```
汇总本周的阅读、搜索、创意记录，生成进度报告。

---

## 自动保存规则

所有输出**自动保存**到 Obsidian，无需手动确认：

| 内容类型 | 保存位置 |
|---------|---------|
| 每日推荐论文 | `Daily Picks/YYYY-MM-DD.md` |
| 搜索结果 | `Search Results/YYYY-MM-DD-{关键词}.md` |
| 论文笔记 | `Paper Notes/{作者}-{年份}-{标题}.md` |
| 头脑风暴 | `Ideation Sessions/YYYY-MM-DD-{主题}.md` |
| 文献综述 | `Literature Reviews/{主题}.md` |
| 概念卡片 | `Concept Cards/{概念名}.md` |

## 研究领域配置

本工作区已针对你的研究方向预配置：

- **核心方向：** AI 素养、自我调节学习、学习分析、智能辅导系统
- **顶期刊列表：** `domains/ai-in-education/journals.md`
- **关键词映射：** `domains/ai-in-education/keyword-mapping.md`（搜索词同义词库）

---

## 典型工作流

```
早上：/daily                           → 看今日推荐论文
发现感兴趣：/summarize {论文}           → 深读 + 自动存笔记
想到新方向：/brainstorm {主题}          → 生成论文创意
深入某方向：/lit-review {主题}          → 系统文献综述
周五：/weekly-report                   → 回顾本周进展
```

---

## 多智能体质量保障

系统内置多个验证 Agent，确保研究质量：

- **Citation Verifier** — 防止幻觉引用
- **Coverage Critic** — 检测文献搜索盲点
- **Summary Auditor** — 验证论文摘要准确性
- **Novelty Checker** — 评估研究方向新颖性
- **Research Planner** — 搜索前澄清研究意图

这些 Agent 在 `/search-papers`、`/summarize`、`/brainstorm` 等命令中**自动触发**，无需手动调用。
